E-Commerce site

Description:

Users can search and browse for products from 4 different categories. They can sort or filter products depending on the price or category, and add or remove products to cart. The checkout includes a billing information form and an order summary as the final step.

Motivation:

The primary motive was to try out and learn both Redux and Material-UI. I also liked the idea of an e-commerce project and discovered Fake Store API to get data for my application. I knew managing application state is an important aspect of a shopping site, so I chose to work with Redux because it is the most widely used library in that regard. I had never utilized an UI framework before, but after some research I realized I cannot go wrong by choosing Material-UI.

Challenges I faced:

Getting accustomed to Redux workflow was rather intimidating at first. It was difficult to wrap my head around all the boilerplate code necessary to even begin doing something. Eventually it did start to make sense. Due to my unfamiliarity with Redux, even simple tasks took a while to implement, but I got there at the end, and saved myself an immense amount of prop drilling.

I had never worked with a styling/UI framework before either. It felt strange not having to build everything from scratch. It took some time to familiarize with the Material-UI documentation and endless list of possibilities regarding each separate component. I quickly learned that one of the most challenging parts is to customize a component exactly the way I want to.

Technologies used:
  • React
  • Redux
  • Material-UI
  • React Router

Recipes app

Description:

Users can search and browse for recipes from a wide array of food categories. Detailed information about each particular recipe with ingredients, instructions, images, videos, articles is included. Possibility to add recipes to favorites and store it for later use.

Motivation:

I desired to practice TypeScript and get more familiar with Redux. I felt that the overall complexity level of the project was just right to put my theoretical TS knowledge to use. I chose to do a project related to food because I found TheMealDB API that fit the bill for my data requirements almost perfectly.

Challenges faced:

Biggest hurdle was in the beginning with all the setup, and getting used to writing everything in TypeScript (TS). I got constantly screamed at by long-winded TS errors, which was rather intimidating at first. For educational purposes, I tried to steer away from using any types to ease out of trouble. Even simple tasks became quite time consuming.

TS did start to grow on me as I got more used to thinking on that level, and I particularly liked the IntelliSense aspect of it. I know more practice is needed for me to be fully comfortable with TS, but I am glad I chose to take the first step.

Technologies used:
  • React
  • TypeScript
  • SCSS
  • React Router
  • Framer Motion
  • Redux

Quiz app

Description:

Users can generate their own custom quiz and play it. They are able to choose the number of questions, category and difficulty level. A result screen is shown at the end of the quiz with a chance to review the answers.

Motivation:

Before I started the project, I identified I had to improve my core knowledge in React and JavaScript to feel more comfortable with them. A quiz app seemed a good option to solidify the fundamental skills. It gave me the possibility to use React Router for the first time, and I also planned to add animations to my project. I found Open Trivia Database API to supply my app with data.

Challenges I faced:

One particular challenging part was when I wished to simultaneously fetch data and redirect the user to a new page after a button click. Initially, I only managed to make one of the two to successfully work. I asked for help from a more experienced developer friend, and the solution was to use React Router useHistory hook and the push method. I had to add the state which included the fetched questions data as an additional parameter  history.push({ pathname: '/questions', state: { questions } }); .

I had never worked with an animations library before, so getting accustomed to Framer Motion was also an interesting learning experience. The documentation is quite extensive and there are a lot of options to choose from. It was time consuming to find exactly what I needed. I learned that tweaking animations to just feel right can be a never ending task.

Technologies used:
  • React
  • SCSS
  • Framer Motion
  • React Router

Wind turbines tracker

Description:

Web app locating wind turbines in Alaska. Users can find detailed information about each wind turbine by clicking on the icon on the map. There is an option to copy coordinates to clipboard as well. The map title includes an overview of all the wind turbines in the region.

Motivation:

I have always loved working with maps, and I knew I wanted to make a project using Google Maps API. Therefore, I searched for interesting data to build my app around. I eventually found wind turbines data from U.S. Geological Survey API which seemed compelling enough.

Challenges faced:

In the beginning, the main challenge was to find data which had geographical coordinates included in the dataset. Later on, I realized that there was a lot of data missing. I had to start filling in the gaps with randomly generated numbers to make the app look more presentable.

This was the first bigger React project I had done, and that brought up many challenges in itself. Integrating React with Google Maps API was the main one. I had a difficult time finding exactly what I needed from the extensive Google Maps API documentation.

Also, I did not really know how to exactly implement most of the things in React. I started to focus on smaller sections of the demo code to get a better understanding what is going on, and eventually it started to make more sense.

Technologies used:
  • React
  • SCSS
  • React Google Maps API

Q&A

When and how did I start learning web development?

Web development sparked my interest during lockdown in May 2020. In the beginning, I followed along Colt Steele's The Web Developer Bootcamp course on Udemy, read articles, and watched other educational videos on the side. I completed Codecademy's JavaScript curriculum in the summer, and then continued working on personal projects until now.

What do I like the most about being a front-end developer?

I love the aspect of getting visual feedback once I change something in code, and knowing that I will have an direct impact on everything the user sees and interacts with. I am also fond of the front-end landscape in general because it is always evolving and changing. I will continually have the opportunity to interact with new tools and learn new skills.

What is usually the most difficult part of a front-end project for me and why?

I have learned that building consistent user experience can be rather challenging at times. Especially in terms of making sure everything is compatible with every browser and looking good on different devices and screen sizes.

Also, choosing the right set of tools for a particular project, and setting everything up properly has been quite tricky and time consuming as well (e.g. Redux, TypeScript). At the same time, I acknowledge that tools are there to help developers so that is the trade-off I guess.

What resources do I use to learn about the latest in front-end development?

I have signed up for different newsletters (e.g. JavaScript Weekly, FreeCodeCamp) and participate in several Discord channels where knowledge is shared, and asking questions is encouraged (e.g. The Odin Project). I also search for articles, videos, and interactive coding lessons on specific topics myself that usually come up while I am working on my personal projects.

What is my personal motto?

If I'm the smartest person in the room, then I'm definitely in the wrong room.

What are my near-term plans for the future?

I have realized that what first started as a hobby could possibly be a very rewarding career. I am interested in an internship or a junior front-end developer job to gain valuable experience with real projects and further develop my skills.
